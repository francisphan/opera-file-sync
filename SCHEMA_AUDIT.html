7 Recommendations Summary

High Priority — Remove / Uninstall
  • Uninstall SigParser package — ~70 fields across Contact and Lead, all 0% usage
  • Remove Google Analytics UTM fields (5 per object) — all 0%, never configured
  • Delete check-in/out dates from Contact (Check_in_date_C__c, Check_out_Date_C__c) — these belong on TVRS_Guest__c only
  • Delete TVRS_Guest__c lookup from Opportunity — 0%, never used
  • Delete TVRS_Guest__c lookup from Contact — 0%, relationship goes the other direction
Medium Priority — Consolidate Duplicates
  • Remove person data from TVRS_Guest__c (name, phone, city, state, country) — accessible via Contact__c lookup
  • Remove Email__c and Phone__c from Opportunity — accessible via Contact/Account relationships
  • Consolidate PoC__c vs Point_of_Contact__c on Opportunity — appears to be the same data in two fields
  • Move person-level booleans off TVRS_Guest__c (In_Conversation, Not_interested, Ready_for_pardot, Follow_up, Ready_for_PV_mail) — keep on Contact only
  • Evaluate Historical_*_Id__c fields on Opportunity — if data migration is long done, these can go
Low Priority — Clean Up When Convenient
  • Clarify or delete ambiguous fields: Selected_1__c, Check__c, Update__c, Created__c, X7F__c, Driver__c
  • Delete confirmed dead fields: Room__c, Comply_Tasting_Date__c, Travel_for_wine__c, CUIT__c, Interested_in_Kosher__c, Investment_Date__c
  • Resolve Replied_to_Mkt_campaign_2025__c — make it year-agnostic or replace for 2026
  • Consider whether Favorite_Sports_Teams__c belongs on Opportunity or Contact
  • Review Language_Text__c on Lead (redundant with Primary_Language__c picklist at 100%)

This audit was generated by querying live Salesforce data via the OPERA sync API credentials. Field usage percentages are based on non-null (or true, for booleans) values across all records in each object.

Textarea fields that can't be filtered in SOQL (Comments, Notes, etc.) could not be checked for usage — these need manual review.
